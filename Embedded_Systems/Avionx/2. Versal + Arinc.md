# What is ARINC-653
Software specification for real-time operating systems in safety-critical avionics. 
Ensures multiple applications can operate simultaneously without interference.
The specification supports space and time partitioning in Integrated Modular Avionics architectures.
Different applications with varying safety levels can run concurrently on the same hardware

Typically:

One CPU (or a set of CPUs)

Multiple partitions

Each partition has its own allocated memory

The CPU runs the partitions one at a time in scheduled time slots

So partitions share the CPU, but do NOT share memory directly.

## How It Actually Works

1. One CPU, Multiple Partitions

The CPU runs several partitions by time slicing. EXAMPLE:
| Time Slot | Running Partition |
| --------- | ----------------- |
| 0–20 ms   | Flight Control    |
| 20–40 ms  | Navigation        |
| 40–60 ms  | Display           |
| 60–80 ms  | Maintenance       |

Then the cycle repeats.

This is called **a major time frame** in ARINC-653.

2. Memory Is Divided

The system allocates separate memory regions for each partition. EXAMPLE
| Partition      | Memory Region |
| -------------- | ------------- |
| Flight Control | 0x0000–0x1FFF |
| Navigation     | 0x2000–0x3FFF |
| Display        | 0x4000–0x5FFF |

Hardware protection (via MMU/MPU) prevents a partition from accessing memory that isn’t assigned to it.
```
CPU
 └── ARINC-653 OS Kernel
      ├── Partition A (Flight Control App)
      │      ├── Process 1
      │      └── Process 2
      │
      ├── Partition B (Navigation App)
      │      ├── Process 1
      │      └── Process 2
      │
      └── Partition C (Display App)
```

# VMK180 Architecture with ARINC-653

The AMD Xilinx Versal Prime VMK180 Evaluation Kit contains a Versal SoC, which includes:
- Application processors (ARM Cortex-A72) APU
- Real-time processors (ARM Cortex-R5F) RPU
- Programmable logic (FPGA fabric)

The ARINC-653 OS is a real-time operating system (RTOS) that runs directly on the processor cores of the board.
In my case, it would normally run on the ARM cores (A72 or R5F), while the FPGA fabric might implement accelerators or I/O logic.

```
Versal SoC

ARM Cortex-A72
   ↓
ARINC-653 RTOS
   ↓
Partitions
   ↓
Processes / Applications (Navigation, Display, Control)
```
```
FPGA Fabric
   ↓
Sensor interfaces / accelerators
```
## ARINC-653 OS
Examples of ARINC-653 RTOS implementations include:
- VxWorks 653
- INTEGRITY-178 tuMP
- Deos
- PikeOS
These run directly on the CPU cores.

Even though it runs on bare metal, it still needs a kernel.

## ARINC-653 Kernel
The **ARINC-653 kernel** is responsible for:
- Partition scheduling (time slicing)
- Memory protection
- Inter-partition communication
- Process scheduling inside partitions
- Health monitoring

So the kernel is what enforces the ARINC-653 rules.

# APEX Interface
In ARINC 653, the Application Executive (APEX) interface is the standard API that applications use to interact with the ARINC 653 operating system.
Think of it as the set of system calls that software inside a partition uses to talk to the ARINC OS kernel.

Why APEX Exists
Aircraft software is developed by many vendors. APEX ensures that:
- Applications don’t depend on a specific RTOS
- The same application can run on different ARINC-653 systems (like VxWorks 653 or INTEGRITY-178 tuMP)
- Safety certification is easier
So APEX provides a standardized programming interface.

```
Hardware
     ↓
ARINC 653 Kernel
     ↓
ARINC 653 OS
     ↓
APEX API - Device Drivers - Configuration Tools
     ↓
Application Code (Navigation, Display, Control)
```

| Category                 | Example Purpose                          |
| ------------------------ | ---------------------------------------- |
| **Process management**   | Create/start/suspend processes (threads) |
| **Time management**      | Delays, periodic execution               |
| **Partition management** | Control partition state                  |
| **Communication**        | Send data to other partitions            |
| **Synchronization**      | Semaphores, events                       |
| **Health monitoring**    | Report faults                            |


# Notes
Partitions Are NOT Separate Operating Systems
In pure ARINC 653 systems:
- Partitions usually contain applications
- They do not run full operating systems
So you would not normally run:
- Linux in one partition
- another RTOS in another partition

Instead you run **applications using the APEX API.**
