# What is an FPGA?
An Field-Programmable Gate Array (FPGA) is a chip that can be configured to become custom digital hardware.

Unlike a CPU that runs instructions sequentially, an FPGA implements hardware circuits that run in parallel. Instead of writing software, you describe hardware using languages like:
- VHDL
- Verilog
After synthesis, the FPGA becomes a custom circuit.

| Device | How it works                        |
| ------ | ----------------------------------- |
| CPU    | Executes instructions sequentially  |
| GPU    | Many parallel processors            |
| FPGA   | Custom hardware circuits you design |

### System Flow = Data Path
```
Sensor
   ↓
FPGA (signal filtering / preprocessing)
   ↓
CPU running ARINC 653
   ↓
Partition: Navigation
   ↓
Partition: Display
```

### System Architecture
```
Hardware (VMK180)
   ├─ CPU cores
   │     └─ ARINC 653 OS
   │           └─ Partitions
   │                └─ Applications using APEX
   │
   └─ FPGA fabric
         └─ Custom hardware accelerators
```

**How the FPGA Actually Talks to the CPU**
In a Versal SoC, the FPGA fabric and CPU communicate through on-chip buses and memory.

Typical interaction:
```
Sensor
   ↓
FPGA logic
   ↓
Shared memory / DMA
   ↓
CPU
   ↓
ARINC 653 partition reads the data
```
The FPGA might:
Write processed data to memory
Trigger an interrupt
The CPU partition reads the result
